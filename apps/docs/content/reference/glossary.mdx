# Glossary

## Applied architecture

Using multiple engineering concepts together inside a recognizable product system like booking, chat, or payments.

## Compare mode

A StackLab pattern where multiple approaches are shown side by side with explicit tradeoffs.

## Hidden page

A public docs page that is reachable by URL but omitted from main navigation.

## Production-oriented template

A starter that explains why it exists, what it includes, what it leaves out, and what tradeoffs it accepts.

## Product pillar

One of the five core StackLab product pillars: Learn, Simulate, Compare, Build, or Contribute.
