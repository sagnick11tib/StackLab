# Core Pillars

Every major feature in StackLab should strengthen at least one of these pillars.

<Columns cols={2}>
  <Card title="Learn">
    Clear explanations, glossaries, summaries, and foundational concept pages.
  </Card>
  <Card title="Simulate">
    Interactive visualizations that show runtime behavior, system flow, scale,
    and failure.
  </Card>
  <Card title="Compare">
    Side-by-side tradeoff exploration for patterns, architectures, and stack
    choices.
  </Card>
  <Card title="Build">
    Production-oriented starter templates and implementation guidance.
  </Card>
  <Card title="Contribute">
    Strong open-source workflows, AI rules, and module standards for growth
    without chaos.
  </Card>
</Columns>

## Pillar Standard

If a future feature does not help users:

- understand a concept
- inspect behavior
- compare options
- start building more safely
- or contribute coherently

it probably does not belong in StackLab.
