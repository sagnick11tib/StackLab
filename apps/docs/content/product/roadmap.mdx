# Roadmap

StackLab is large by design, so it has to grow in phases.

## Phase 0: Foundation

- product vision
- canonical docs
- design language
- contribution rules
- information architecture

## Phase 1: Public Surface

- production Nextra docs app on the docs subdomain
- introduction and product pages
- labs and templates overview pages
- contributor guidance
- public roadmap and brand system

## Phase 2: First Interactive Labs

- JavaScript runtime lab
- system design flow lab
- auth and RBAC compare module
- booking concurrency lab

## Phase 3: Template Foundation

- first Next.js templates
- first MERN or PERN starter
- template rationale pages
- linked implementation notes

## Phase 4: Product Expansion

- applied architecture labs
- richer compare mode
- saved scenarios
- broader contributor workflows

## Phase 5: Ecosystem Growth

- community-created modules
- role-based learning paths
- challenge and interview modes
- broader stack coverage

## Roadmap Rule

The roadmap should optimize for coherence and quality, not raw feature count.
